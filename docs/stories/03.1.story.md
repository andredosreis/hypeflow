## User Story

> Como **agência a operar campanhas de captação em múltiplas plataformas**, quero **um hub central de ingestão de leads onde cada provider (Meta, Evolution API, Tally, Typeform, ManyChat, n8n…) entrega eventos via um contrato universal, com deduplicação write-time, dead-letter queue para payloads inválidos e rastreabilidade de origem completa**, para que **(a) leads não sejam duplicados nem perdidos silenciosamente, (b) o ROI por canal seja calculável a partir do `source.platform` em cada lead, e (c) ligar um novo provider seja apenas escrever um adapter sem tocar no core do hub**.

---

## Contexto

O [FDD Lead Ingestion Hub 1.0](../fdd/lead-ingestion-hub-fdd.md) define a feature em detalhe. Esta story implementa o **MVP**: o pipeline completo de ingestão (auth → rate-limit → adapter → dedup → score-stub → persist → publish), o contrato universal Lead DTO, e **um adapter — Evolution API** — como prova de que o contrato funciona end-to-end. Adapters Meta/Tally/Typeform/ManyChat/n8n ficam para stories subsequentes (FDD diz explicitamente "FDD separado por provider").

### Estado atual

- `app/api/webhooks/ghl/route.ts` é o único webhook em produção; é ad-hoc, não segue o contrato universal, persiste directamente em `leads` sem dedup ou dead-letter, usa service role sem isolamento.
- Tabela `leads` (migração `0001_initial_schema.sql:97-126`) tem campos básicos mas **falta**: `event_id` para idempotência, `metadata` JSONB, `email_normalized`, `phone_normalized`, `source_platform` distinto de `source`, `raw_payload` para auditoria, `schema_version`.
- Tabela `webhook_failures` **não existe** — sem dead-letter queue, falhas de schema são descartadas silenciosamente.
- `libphonenumber-js` **não está instalado** — sem normalização E.164 a dedup por phone é frágil.

### Non-negotiables do FDD aplicados ao MVP

1. `source.platform` preenchido em 100 % dos leads persistidos (FDD §2 H6).
2. `agency_id` derivado pelo hub a partir do `client_id`, **nunca** vem do provider (FDD §1 invariante + §6 Invariantes).
3. PII (`email`, `phone`, `name`) **nunca** em logs em texto claro — apenas hashes (FDD §6 Invariantes + §7 Regras de mascaramento).
4. `raw_payload` apenas em `webhook_failures`, **nunca** em logs (FDD §7).
5. Provider que envia `client_id` inexistente recebe **200 silencioso** (não 404) para não vazar quais client_ids são válidos (FDD §4 passo 4).
6. HMAC/token inválido → **401 sem detalhes**, payload **descartado permanentemente** (não vai para dead-letter — possível ataque) (FDD §6).
7. Score Engine **fail-open** — falha do scoring não reverte a persistência da lead (FDD §6).

### Trade-off — Score Engine fica como stub nesta story

O FDD trata Score Engine como dependência externa do hub. Implementar o cálculo determinístico de score com `score_weights` por agência é trabalho separado (relacionado com `score-engine-fdd.md` se existir, ou story futura). Nesta MVP o `score = 0` é o resultado canónico, com `metadata.score_pending = true` no lead persistido. Quando o Score Engine real ficar pronto, processa estes leads em batch ou em tempo real.

### Fluxo do route handler (pseudocode)

```ts
export async function POST(req: NextRequest, { params }: { params: Promise<{ client_id: string }> }) {
  const { client_id } = await params
  const start = Date.now()

  // 0. Size guard
  if (Number(req.headers.get('content-length') ?? '0') > 1_048_576) {
    return NextResponse.json({ error: 'Payload too large' }, { status: 413 })
  }

  // 1. Read raw body once
  const rawText = await req.text()

  // 2. Auth (token from env keyed by client_id)
  const expectedToken = process.env[`EVOLUTION_TOKEN_${client_id.toUpperCase().replace(/-/g, '_')}`]
  if (!expectedToken || !evolutionAdapter.verify(req, expectedToken)) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
  }

  // 3. Lookup client → derive agency_id (service role; client_id may not exist)
  const service = await createServiceClient()
  const { data: client } = await service
    .from('clients')
    .select('id, agency_id')
    .eq('id', client_id)
    .maybeSingle()
  if (!client) {
    // 200 silent — do not leak whether client_id is valid
    return NextResponse.json({ ok: true })
  }

  // 4. Rate limit (after client lookup so we can bucket by agency)
  const ip = getClientIp(req)
  const rl = await enforceWebhookRateLimit({ ip, agencyId: client.agency_id })
  if (!rl.allowed) {
    return NextResponse.json({ error: 'Rate limit exceeded' }, {
      status: 429,
      headers: { 'Retry-After': String(rl.retryAfter) },
    })
  }

  // 5. Parse JSON
  let body: unknown
  try { body = JSON.parse(rawText) } catch { return NextResponse.json({ error: 'Invalid JSON' }, { status: 400 }) }

  // 6. Adapter parse → LeadDTO or { skip: true }
  let dto: LeadDTO | { skip: true }
  try {
    dto = await evolutionAdapter.parse(body, { clientId: client.id, rawText })
  } catch (err) {
    // Schema invalid → dead-letter + 200 to provider (provider stops retrying)
    await recordWebhookFailure(service, {
      provider: 'evolution',
      clientId: client.id,
      agencyId: client.agency_id,
      rawPayload: body as object,
      reason: 'schema_invalid',
      errorDetail: String(err),
    })
    return NextResponse.json({ ok: true })
  }
  if ('skip' in dto) {
    return NextResponse.json({ ok: true })
  }

  // 7. Validate DTO with Zod (paranoid double-check)
  const parsed = leadDtoSchema.safeParse(dto)
  if (!parsed.success) {
    await recordWebhookFailure(service, {
      provider: 'evolution', clientId: client.id, agencyId: client.agency_id,
      rawPayload: body as object, reason: 'dto_validation_failed',
      errorDetail: JSON.stringify(parsed.error.flatten()),
    })
    return NextResponse.json({ ok: true })
  }
  const validDto = parsed.data

  // 8. Dedup
  const dupe = await findDuplicate(service, {
    clientId: client.id,
    eventId: validDto.event_id,
    emailNormalized: validDto.contact.email ? normalizeEmail(validDto.contact.email) : null,
    phoneNormalized: validDto.contact.phone ? normalizePhone(validDto.contact.phone) : null,
  })

  // 9. Score (stub)
  const score = scoreLeadStub(validDto)

  // 10. Persist (handles all dedup branches)
  try {
    const result = await persistLead(service, validDto, client.agency_id, dupe, score)
    logIngestionEvent({
      level: 'info',
      event: result.isNew ? 'lead.created' : 'lead.merged',
      dto: validDto,
      durationMs: Date.now() - start,
    })
    return NextResponse.json({ ok: true, event_id: validDto.event_id })
  } catch (err) {
    // DB unavailable → dead-letter + 503 (provider retries)
    await recordWebhookFailure(service, {
      provider: 'evolution', clientId: client.id, agencyId: client.agency_id,
      rawPayload: body as object, reason: 'db_unavailable',
      errorDetail: String(err),
    })
    return NextResponse.json({ error: 'Service unavailable' }, {
      status: 503,
      headers: { 'Retry-After': '30' },
    })
  }
}
```

-- Story 03.1 / FDD lead-ingestion-hub-fdd.md

begin;

alter table public.leads
  add column if not exists event_id text,
  add column if not exists metadata jsonb not null default '{}',
  add column if not exists raw_payload jsonb,
  add column if not exists email_normalized text,
  add column if not exists phone_normalized text,
  add column if not exists source_platform text,
  add column if not exists schema_version text not null default 'v1',
  add column if not exists provider text;

-- Backfill (safe: nullable columns, no constraint)
update public.leads set source_platform = source where source_platform is null;
update public.leads set provider = 'manual' where provider is null;
update public.leads set email_normalized = lower(trim(email)) where email is not null and email_normalized is null;

-- Partial unique indexes (don't break manual leads without email/phone)
create unique index if not exists leads_event_id_uniq
  on public.leads (event_id) where event_id is not null;
create unique index if not exists leads_client_email_normalized_uniq
  on public.leads (client_id, email_normalized) where email_normalized is not null;
create unique index if not exists leads_client_phone_normalized_uniq
  on public.leads (client_id, phone_normalized) where phone_normalized is not null;

create index if not exists leads_received_recent
  on public.leads (client_id, created_at desc);

-- Dead-letter queue
create table public.webhook_failures (
  id              uuid primary key default gen_random_uuid(),
  provider        text not null,
  client_id       uuid references public.clients(id) on delete cascade,
  agency_id       uuid references public.agencies(id) on delete cascade,
