## User Story

> Como **equipa de desenvolvimento**, quero **eliminar o `createNullClient` e ter um mock de Supabase com tipagem completa e detecção de erros em runtime**, para que **chamadas à API do Supabase em modo preview/demo falhem de forma visível em vez de passar silenciosas e mascarar bugs**.

---

## Contexto

A auditoria [context7-audit-report.md](../architecture/context7-audit-report.md) (v2, 2026-04-21) identificou o achado **CR-02**: o mecanismo `createNullClient` em `apps/hypeflow/lib/supabase/server.ts:6-36` tem três problemas graves:

1. **API incompleta:** implementa apenas `auth`, `from` e `channel`. Faltam `rpc`, `storage`, `functions`, `removeChannel`, entre outros. Qualquer chamada fora da superfície implementada explode em `TypeError` em runtime.
2. **`.then` falso (linhas 26-27):** faz o `noopQuery` parecer uma Promise. Código que faz `await db.from(...).select(...)` funciona sem erro, mas retorna sempre `{ data: null, error: null }` — bug silencioso que pode passar a produção sem ser notado em preview.
3. **Tipagem `any`:** elimina verificação de TypeScript em toda a árvore de consumidores do mock.

O objectivo desta story é substituir o mock frágil por uma solução real: **Supabase local** para dev/preview (primeira escolha) e **MSW** para testes (segunda escolha).

## Technical Notes

### Estrutura preferida

```
hypeflow-os/
├── supabase/
│   ├── config.toml              ← supabase start
│   ├── migrations/
│   └── seed.sql                 ← dados demo
├── packages/
│   └── database/
│       └── types.ts             ← gerado por supabase gen types
└── test/
    └── msw/
        ├── handlers/
        │   └── supabase.ts      ← intercepta /rest/v1/* do Supabase
        └── server.ts
```

### Fail-fast exemplo

```ts
// apps/hypeflow/lib/supabase/server.ts
const SUPABASE_URL = process.env.NEXT_PUBLIC_SUPABASE_URL
const SUPABASE_ANON = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY

if (!SUPABASE_URL || !SUPABASE_ANON) {
  throw new Error(
    'Missing SUPABASE env vars. ' +
    'Run `supabase start` for local dev or set NEXT_PUBLIC_SUPABASE_URL and ' +
    'NEXT_PUBLIC_SUPABASE_ANON_KEY in .env.local. See README.md.'
  )
}

export async function createClient() {
  const cookieStore = await cookies()
  return createServerClient<Database>(SUPABASE_URL, SUPABASE_ANON, { /* ... */ })
}
```

### Trade-off assumido

Esta story **trade-offs deliberadamente** o modo "demo público sem DB" em troca de:
- Tipagem completa em toda a árvore
- Detecção de erros em runtime (nenhum método silenciado)
- Um único caminho de código em dev/preview/produção

Se o modo demo público voltar a ser necessário (ex.: landing page com dashboard fake), será abordado numa story separada com Repository Pattern por cima do Supabase client.
