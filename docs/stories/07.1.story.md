## User Story

> Como **gestor da agência**, quero **um motor de automações nativo na plataforma, com integrações para N8N, Make e ManyChat** para que **acções repetitivas sejam executadas 24/7 pelo próprio sistema, podendo extender para ferramentas externas quando necessário**.

---

## Contexto

A HYPE Flow vende "motor comercial 24/7 automático" — portanto, automações não são um feature opcional, são o produto core.

**Hierarquia de automações:**

1. **Motor Nativo (CORE)** — O sistema tem a sua própria engine de automações. É o produto principal. Regras visuais, triggers, condições e acções executadas directamente pela plataforma sem dependência de ferramentas externas.
2. **N8N** — Extensão para fluxos complexos. Webhook bidireccional com self-hosted N8N.
3. **Make (ex-Integromat)** — Extensão para clientes/agências que já usam Make.
4. **ManyChat** — Integração para automações de conversas (Instagram DMs, Facebook Messenger, WhatsApp) via ManyChat flows conectados ao CRM da plataforma.

## Acceptance Criteria

### Motor de Automações Nativo

- [ ] **AC1:** Admin pode criar regras de automação em `/config/automacoes` com editor visual
- [ ] **AC2:** **Triggers disponíveis:**
  - Lead entra numa fase específica
  - Lead sem contacto há X horas/dias
  - Lead criada de canal específico (ex: só Facebook)
  - Score de lead muda (sobe acima de Y)
  - Call agendada
  - Call não atendida (no-show)
  - Lead marcada como perdida
  - Novo cliente adicionado
- [ ] **AC3:** **Condições (filtros):**
  - Fonte da lead é / não é
  - Score maior / menor que
  - Fase actual é / não é
  - Agente atribuído é / não é
  - Temperatura é
- [ ] **AC4:** **Acções disponíveis:**
  - Enviar mensagem WhatsApp (template)
  - Enviar email (template)
  - Mover lead para fase
  - Atribuir agente
  - Adicionar tag
  - Criar task de follow-up
  - Enviar webhook (para N8N / Make / custom)
  - Enviar notificação in-app ao agente
- [ ] **AC5:** Cada automação tem: nome, ativa/pausada, logs de execuções, contador de execuções
- [ ] **AC6:** Delay configurável nas acções (ex: "2 horas após trigger")
- [ ] **AC7:** Automações podem ser globais (toda a agência) ou por cliente

### N8N Integration

- [ ] **AC8:** Admin configura URL do N8N self-hosted + API key em `/config/integracoes`
- [ ] **AC9:** A plataforma pode **enviar webhooks para N8N** ao disparar qualquer trigger (Trigger → N8N Workflow)
- [ ] **AC10:** N8N pode **chamar a API da plataforma** para executar acções (N8N → REST API)
- [ ] **AC11:** Endpoint REST autenticado para N8N: `POST /api/n8n/actions`
  - Acções suportadas: `update_lead`, `move_stage`, `assign_agent`, `add_tag`, `create_call`, `send_notification`
- [ ] **AC12:** Página de "N8N Webhooks" mostra todos os webhooks enviados + status de resposta (log)
- [ ] **AC13:** Templates de workflows N8N para download (JSON) — ex: "Reactivação WhatsApp", "Follow-up automático"

### Make (Integromat) Integration

- [ ] **AC14:** Admin configura Make webhook URL em `/config/integracoes`
- [ ] **AC15:** A plataforma envia eventos para Make via webhook (mesma lógica do N8N)
- [ ] **AC16:** Endpoint REST autenticado para Make: `POST /api/make/actions` (mesmas acções do N8N)
- [ ] **AC17:** Templates de cenários Make para download — ex: "Lead Facebook → WhatsApp → CRM"

### ManyChat Integration

- [ ] **AC18:** Admin conecta conta ManyChat via API key em `/config/integracoes`
- [ ] **AC19:** Quando lead entra no CRM via ManyChat (Instagram DM, FB Messenger, WhatsApp) → é criada automaticamente na plataforma com `source = 'manychat'`
- [ ] **AC20:** A plataforma pode **disparar fluxos ManyChat** como acção de automação (ex: "Lead entra em fase Qualificada → iniciar flow ManyChat 'Boas Vindas'")
- [ ] **AC21:** ManyChat pode **actualizar dados no CRM** via webhook (ex: utilizador respondeu → lead temperature sobe para warm)
- [ ] **AC22:** Sincronização de campos: nome, email, telefone, tags ManyChat ↔ campos da lead
- [ ] **AC23:** View de leads com origem ManyChat mostra o flow de onde vieram

### Logs & Monitoring

- [ ] **AC24:** Log de todas as automações executadas: data, trigger, acções, resultado (sucesso/erro)
- [ ] **AC25:** Alerta quando automação falha 3x seguidas → notificação ao admin
- [ ] **AC26:** Dashboard de automações: execuções hoje, taxa de sucesso, top 5 automações mais activas
- [ ] **AC27:** Separação visual no dashboard: "Automações Nativas" vs "Via N8N" vs "Via Make" vs "Via ManyChat"

## Technical Notes

### Arquitectura do Motor de Automações

```
Trigger ocorre (DB change ou scheduled check)
         │
         ▼
Supabase Edge Function: automation-engine
         │
    Carregar regras activas que matcham o trigger
         │
    Avaliar condições (filtros)
         │
    Executar acções em sequência (com delay se configurado)
         │
    Guardar log em automation_logs
```

### Schema de Automações

```sql
CREATE TABLE automation_rules (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  agency_id     UUID REFERENCES agencies(id),
  client_id     UUID REFERENCES clients(id), -- NULL = global
  name          TEXT NOT NULL,
  is_active     BOOLEAN DEFAULT TRUE,
  trigger_type  TEXT NOT NULL,  -- lead_stage_changed, lead_no_contact, etc.
  trigger_config JSONB DEFAULT '{}',  -- config específica do trigger
  conditions    JSONB DEFAULT '[]',   -- array de condições
  actions       JSONB DEFAULT '[]',   -- array de acções com delay
  execution_count INTEGER DEFAULT 0,
  last_executed_at TIMESTAMPTZ,
  created_at    TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE automation_logs (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  rule_id         UUID REFERENCES automation_rules(id),
  agency_id       UUID REFERENCES agencies(id),
  lead_id         UUID REFERENCES leads(id),
  trigger_data    JSONB,
  actions_executed JSONB,
  status          TEXT, -- success, partial_failure, failed
  error_message   TEXT,
  executed_at     TIMESTAMPTZ DEFAULT NOW()
);
```

### Exemplo de regra em JSON

```json
{
  "trigger_type": "lead_no_contact",
  "trigger_config": { "hours": 24 },
  "conditions": [
    { "field": "temperature", "operator": "eq", "value": "hot" }
  ],
  "actions": [
    {
      "type": "send_whatsapp",
      "delay_hours": 0,
      "config": { "template_id": "followup-hot-lead" }
    },
    {
      "type": "send_webhook",
      "delay_hours": 1,
      "config": { "url": "https://n8n.hypeflow.pt/webhook/lead-no-contact" }
    }
  ]
}
```

### N8N / Make Webhook Payload

```typescript
// Payload enviado para N8N / Make ao disparar trigger:
{
  event: 'lead_stage_changed',
  timestamp: '2026-04-06T10:00:00Z',
  agency_id: 'uuid',
  client_id: 'uuid',
  lead: {
    id: 'uuid',
    full_name: 'João Silva',
    email: 'joao@email.com',
    phone: '+351912345678',
    source: 'facebook',
    score: 85,
    temperature: 'hot',
    previous_stage: 'contacted',
    current_stage: 'scheduled',
  }
}
```

### API Endpoint para N8N / Make chamar de volta

```typescript
// POST /api/n8n/actions
// Headers: Authorization: Bearer {api_key}
// Body:
{
  action: 'update_lead',
  lead_id: 'uuid',
  data: { status: 'qualified', score: 90, tags: ['high-priority'] }
}
```
