## User Story

> Como **gestor comercial**, quero **integração completa bidirecional com Google Calendar e Google Meet** para que **todos os agendamentos sejam criados, sincronizados e actualizados em tempo real** entre a plataforma e o Google Calendar da equipa.

---

## Contexto

A HYPE Flow usa Google Calendar e Google Meet como ferramentas de calls de vendas. O utilizador já tem um link de scheduling configurado (`calendar.app.google/NQFPHeXBQ7oqWswi8`), mas quer integração completa — criação de eventos programática, sincronização em tempo real, Meet link automático, e sincronização bidirecional (mudanças no Google reflectem no sistema e vice-versa).

**Abordagem:** Integração completa via Google Calendar API v3 + Google Meet (via conferenceData) + Google OAuth 2.0 com refresh tokens + Webhooks push notifications para tempo real.

---

## Acceptance Criteria

### OAuth & Setup
- [ ] **AC1:** Admin conecta conta Google da equipa via OAuth 2.0 em `/config/integracoes`
- [ ] **AC2:** Scopes solicitados: `calendar`, `calendar.events`, `calendar.settings.readonly`
- [ ] **AC3:** Refresh token guardado encriptado — renovação automática sem re-autenticação
- [ ] **AC4:** Selector de qual Google Calendar usar (a conta pode ter múltiplos calendários)

### Criação de Calls
- [ ] **AC5:** Ao agendar call na plataforma, é criado evento no Google Calendar automaticamente
- [ ] **AC6:** Evento criado com `conferenceData` (Google Meet link gerado automaticamente)
- [ ] **AC7:** Lead/cliente é adicionado como convidado no evento (recebe convite Google)
- [ ] **AC8:** Evento contém: título, descrição com contexto da lead, data/hora, duração, agentes

### Sincronização em Tempo Real
- [ ] **AC9:** Google Calendar Webhooks (push notifications) configurados para receber mudanças
- [ ] **AC10:** Se evento é cancelado no Google Calendar → call actualizada para `cancelled` na plataforma
- [ ] **AC11:** Se evento é reagendado no Google Calendar → `scheduled_at` actualizado na plataforma
- [ ] **AC12:** Se convidado aceita/recusa → estado do convite actualizado na call

### Sincronização Bidirecional
- [ ] **AC13:** Mudanças feitas na plataforma (reagendar, cancelar) actualizam o evento no Google Calendar
- [ ] **AC14:** Calls importadas do Google Calendar (eventos existentes) podem ser associadas a leads
- [ ] **AC15:** Sincronização inicial: importar eventos dos próximos 30 dias do calendário

### Dados em Tempo Real
- [ ] **AC16:** Status das calls actualizado em tempo real na UI (via Supabase Realtime)
- [ ] **AC17:** Meet link visível na plataforma, na notificação email, e no portal do cliente

## Technical Notes

### Google OAuth 2.0 Setup

```typescript
// packages/integrations/google/auth.ts
const GOOGLE_SCOPES = [
  'https://www.googleapis.com/auth/calendar',
  'https://www.googleapis.com/auth/calendar.events',
  'https://www.googleapis.com/auth/calendar.settings.readonly',
  'https://www.googleapis.com/auth/userinfo.email',
]
```

### Criar Evento com Meet

```typescript
// packages/integrations/google/calendar.ts
const event = {
  summary: `Call HYPE Flow — ${lead.full_name}`,
  description: `Lead: ${lead.full_name}\nFonte: ${lead.source}\nScore: ${lead.score}`,
  start: { dateTime: scheduledAt, timeZone: 'Europe/Lisbon' },
  end: { dateTime: endAt, timeZone: 'Europe/Lisbon' },
  attendees: [
    { email: agent.email },
    { email: lead.email },
  ],
  conferenceData: {
    createRequest: {
      requestId: `hypeflow-${callId}`,
      conferenceSolutionKey: { type: 'hangoutsMeet' },
    },
  },
  reminders: {
    useDefault: false,
    overrides: [
      { method: 'email', minutes: 60 },
      { method: 'popup', minutes: 15 },
    ],
  },
}

// POST https://www.googleapis.com/calendar/v3/calendars/{calendarId}/events
// Query param: conferenceDataVersion=1
```

### Webhook Push Notifications

```typescript
// POST https://www.googleapis.com/calendar/v3/calendars/{calendarId}/events/watch
// Payload:
{
  id: 'hypeflow-channel-{agencyId}',
  type: 'web_hook',
  address: 'https://hypeflow-os.vercel.app/api/webhooks/google-calendar',
  expiration: Date.now() + 7 * 24 * 60 * 60 * 1000, // 7 dias (máximo Google)
}
// Webhook deve ser renovado antes de expirar (cron diário de verificação)
```

### Supabase Realtime
