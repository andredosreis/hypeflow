## User Story

> Como **gestor de tráfego**, quero **conectar a conta Meta Business de um cliente** para que **as métricas de Facebook Ads e Instagram Ads sincronizem automaticamente na plataforma**.

---

## Contexto

A HYPE Flow gere campanhas de Meta Ads (Facebook + Instagram) para os seus clientes. Actualmente, os dados são consultados manualmente no Meta Business Suite. Esta story implementa a autenticação OAuth com Meta e a sincronização automática de métricas de campanhas para a tabela `traffic_metrics`.

---

## Acceptance Criteria

- [ ] **AC1:** Admin pode iniciar o fluxo OAuth com Meta Business na ficha de um cliente (`/clientes/{id}/integracoes`)
- [ ] **AC2:** Após OAuth, a integração fica guardada na tabela `integrations` com status `active`
- [ ] **AC3:** O sistema faz sync automático de métricas a cada 6 horas via Edge Function
- [ ] **AC4:** As métricas sincronizadas incluem: impressões, cliques, leads, gastos, CPL, CTR, ROAS por campanha por dia
- [ ] **AC5:** Se o token expirar, o status muda para `expired` e é mostrado alerta na UI
- [ ] **AC6:** Admin pode desconectar a integração (revoga token, status → `disconnected`)
- [ ] **AC7:** Os dados aparecem no dashboard de tráfego filtrados por `platform = 'meta'`
- [ ] **AC8:** Erros de sync ficam registados no campo `error_message` da tabela `integrations`

## Technical Notes

### Integração Meta Marketing API

**Permissões OAuth necessárias:**
```
ads_read
ads_management
business_management
pages_read_engagement
instagram_basic
```

**Endpoints a usar:**
```
GET /{ad-account-id}/insights
  ?fields=impressions,clicks,spend,actions,cost_per_action_type,ctr,cpm
  &time_range={since, until}
  &level=campaign
  &time_increment=1
```

**Edge Function (Supabase):**
```typescript
// supabase/functions/sync-meta-ads/index.ts
// Cron: a cada 6 horas
// Input: integrations com provider = 'meta' e status = 'active'
// Output: upsert em traffic_metrics
```

### Schema relevante

```sql
-- integrations: guardar token OAuth
-- traffic_metrics: normalizar dados Meta
-- ad_campaigns: guardar campanhas Meta com external_id
```

### Variáveis de ambiente necessárias
```
META_APP_ID=
META_APP_SECRET=
META_REDIRECT_URI=
```
